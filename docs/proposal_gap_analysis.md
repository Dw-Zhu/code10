# 开题报告落实差异说明

## 已经落实的核心内容

- 多源公开数据采集与结构化预处理已经落地。
- KMeans 用户分群已经实现并输出聚类结果。
- 逻辑回归自动标注已经实现，并写入评估结果。
- 协同过滤传统推荐算法已经实现，作为对比对象使用。
- 随机森林优化算法已经实现，并输出特征重要性。
- 准确率、精确率、召回率、F1 值等评估结果已经生成。
- 中文图表、关键步骤截图、HTML 报告、Word 报告已经生成。

## 尚未完整落地或没有直接使用的内容

### 1. PySpark 分布式预处理

开题报告写的是“基于 PySpark 对数据进行分布式清洗、转换、整合”。  
当前项目实际使用的是 `pandas + scikit-learn`，没有引入 PySpark，也没有做分布式计算。

### 2. 用户画像数据库

开题报告提到“最终形成完整的用户画像数据库”。  
当前实现是批处理实验流程，结果落在 CSV、JSON、HTML、DOCX 文件中，没有单独搭建数据库或画像服务。

### 3. 标签动态更新机制

开题报告写了“自动标注与动态更新”。  
当前只实现了离线批量重跑后的重新标注，没有做实时更新、增量训练或定时调度。

### 4. 冷启动与稀疏性专项方案

开题报告强调“解决传统协同过滤算法的冷启动与稀疏性问题”。  
当前优化主要体现在结构化特征补充和随机森林重排序上，没有单独做冷启动专项模块，也没有设计专门的冷启动对照实验。

### 5. 文献研究法与文献综述产物

开题报告包含“文献研究法”和参考文献目录。  
当前代码仓库没有单独整理文献综述、引用摘要或理论部分文稿，只保留了实验实现与结果。

### 6. 论文阶段性写作任务

开题报告里的“论文初稿、论文定稿、答辩准备”属于论文过程任务。  
当前项目只交付实验代码、数据、图表和结果报告，没有扩展成完整论文正文。

## 结论

如果按“实验系统是否跑通”来衡量，开题报告里的核心算法链路已经基本落实。  
如果按“论文与工程交付是否完全等同于开题报告原计划”来衡量，还缺的是 PySpark、数据库化、动态更新、冷启动专项设计和论文正文部分。
